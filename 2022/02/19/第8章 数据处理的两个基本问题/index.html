## 第八章 数据处理的两个基本问题

8.1 bx、si、di 和bp

可以这么理解 bx bp 为A类 si di为B类 idata为C类
A、B、C 三类可以随意组合， 但A、B不能同时拥有两个同类。
例如
bx + bp si + di 是错误的
bx + si， bx + di + idata 是正确的

如果[…] 中含有bp, 则段地址默认在ss中

8.3 汇编语言中数据位置的表达

立即数

对于直接包含在机器指令中的数据（执行前在CPU的指令缓存器中）,在汇编语言中成为立即数（idata）,z在汇编中直接给出。
例如： mov ax, 1

寄存器

指令要处理的指令在寄存器上中，在汇编指令中给出相应的寄存器名
例：move ax,bx

段地址（SA）和偏移地址(EA)

指令要处理的数据在内存中，在汇编指令中可以用[x]的格式给好处EA，SA存在某个段寄存器中

  1. 以下默认在ds段寄存器中：
    mov ax, [0]
    mov ax, [di]
    mov ax, [bx]
    mov ax, [bx+si]
    mov ax, [bx+si + 8]
  2. 以下默认在ss段寄存器中:
    mov ax,[bp]
    mov ax,[bp+8]
    mov ax,[bp+si]
    mov ax,[bp+si+8]

8.5 指令要处理的数据有多长

8086CPU的指令，可以处理两个尺寸的数据，分别为byte和word, 分别为8位和16位

  1. 通过寄存器名称来指名处理数据的尺寸
    例如：
    mov ax, 1 处理为16位， 为一个字
    mov al, 1 处理为8位， 为一个字节
    mov al, ds:[0] 一个字节

  2. 在没有寄存器名称的情况下，用操作符 X ptr指名内存单元的长度， X在汇编指令中可以为word活byte
    以下指令用word ptr指令访问内存单元是一个字:
    mov word ptr ds:[0],1
    inc word ptr [bx]
    以下指令用byte ptr指令访问内存单元是一个字节
    mov byte ptr ds:[0],1

  3. 其他情况
    有些指令默认访问的是字节还是字单元，比如 push[1000h] 就不用指明是字还是字节单元，因为push只能进行字操作

8.7 div指令

div是除法指令，使用div做除法的时候应该注意一下几个问题

  1. 除数： 有8位和16位两种
  2. 被除数：默认放在AX或 DX和AX 中，
    1. 如果除数为8位，则被除数为16位。默认在AX中
    2. 如果除数为16位，则被除数在DX和AX中， AX中放低16位 DX放高16位
  3. 结果：
    1. 如果除数为8位， 则AL为商 AH为余数
    2. 如果除数为16位，则AX为商 DX为余数

div 操作的值在哪里，被除数多少位，都掌握在除数手中

8.8 伪指令dd

db: 定义字节类型数据
dw: 定义字类型数据
dd: 定义两个字类型的数据(double word)

8.9 dup

dup是个操作符，在汇编语言中同db、dw、dd等一样，也是由编译器识别处理的符号，需要配合db、dw、dd使用，进行数据的重复

db 3 dup (0) 定义了是3个字节，他们都是0 相当于db 0, 0 ,0
db 3 dup (0,1,2) 定义了是9个字节，他们都是0 相当于db 0, 1 ,2, 0, 1 ,2, 0, 1 ,2
