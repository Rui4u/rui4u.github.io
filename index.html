信号量Signal

分类

  1. SIGABRT (Abort):
    • 用于表示程序调用了 abort() 函数，通常是在出现严重错误时强制终止程序。
    • 可能由应用程序自身或系统发起。
  2. SIGBUS:
    • 表示非法内存访问，如对非法地址进行读写操作。
    • 可能由硬件错误或编程错误引起。
  3. SIGFPE (Floating-Point Exception):
    • 表示浮点运算异常，如除以零或计算结果无穷大。
    • 通常与浮点数计算相关。
  4. SIGILL (Illegal Instruction):
    • 表示执行了非法或未定义的指令。
    • 可能由二进制文件损坏或使用错误的机器码引起。
  5. SIGPIPE:
    • 表示管道或套接字被关闭，但仍然进行写操作。
    • 通常发生在使用网络或进程间通信时。
  6. SIGSEGV (Segmentation Violation):
    • 表示非法内存访问，如空指针访问、已释放内存访问等。
    • 是一种常见的崩溃类型。
  7. SIGSYS (Bad System Call):
    • 表示执行了未定义的系统调用。
    • 通常与底层系统相关。
  8. SIGTRAP (Trap):
    • 表示触发了断点或调试陷阱。
    • 在调试过程中使用。

为什么捕捉Mach异常还需要捕捉UNIX信号:

在 macOS 和 iOS 等系统中，Mach 异常和 UNIX 信号是两种不同的异常处理机制，分别用于处理不同类型的异常。

  1. Mach 异常：
    • Mach 是 macOS 和 iOS 等系统的内核，它负责管理进程间通信和系统调用等底层操作。
    • Mach 异常是由内核发出的，用于处理诸如内存访问错误、非法指令等底层异常。
    • Mach 异常处理通过使用 Mach 线程异常处理器来捕捉和处理异常。可以使用 task_set_exception_ports() 函数设置 Mach 异常的处理回调函数。
  2. UNIX 信号：
    • UNIX 信号是用于进程间通信和处理高层次事件的机制，例如用户按下 Ctrl + C 时产生的 SIGINT 信号。
    • UNIX 信号可以用于捕捉和处理一些高级事件，如程序终止请求、键盘中断等。
    • UNIX 信号处理通过使用 signal() 或 sigaction() 函数来设置信号处理器回调函数。

由于 Mach 异常处理机制是底层的异常处理机制，而 UNIX 信号是高层的进程间通信和事件处理机制，因此在实际应用中，捕捉 Mach 异常和 UNIX 信号可以用于处理不同类型的异常和事件。一些应用场景可能需要同时处理底层的 Mach 异常和高级的 UNIX 信号。

例如，在一个应用程序中，可能需要捕捉 Mach 异常来处理底层的内存访问错误或非法指令，以确保程序在出现底层异常时可以正常退出或处理异常。同时，也可能需要捕捉 UNIX 信号来处理用户的交互操作，如终止应用程序或执行特定操作等。

因此，捕捉 Mach 异常和 UNIX 信号是为了实现全面的异常处理和事件处理，使得应用程序能够在出现各种异常和事件时做出相应的处理和响应。同时，需要注意两种处理机制的使用方式和区别，以确保异常处理和事件处理的准确性和稳定性。

首先要明确的一点是，Mach异常和UNIX信号都可以被捕获，他们也几乎一一对应，但我们需要优先处理Mach异常，因为Mach异常更接近底层，而UNIX信号存在被Mach默认异常处理函数直接退出进程而无法生成的情况。

有一些异常如EXC_CRASH，在Mach异常的阶段没有捕捉，而是放到UNIX信号中捕捉，其中的解释在PLCrashReporter的注释中有详细的解释：

UNIX信号

Mach已经通过异常机制提供了底层的异常处理，但为了兼容更为流行的POSIX标准，BSD在Mach异常机制之上构建的UNIX信号处理机制。异常信号首先被转换为Mach异常，如果没有被外界捕捉，则会被默认的异常处理ux_exception()转换为UNIX信号。
例如SIGSEGV（非法内存访问）转换为 EXC_BAD_ACCESS

前提: 乘数的两个数，位数相等，要不都是8位 要不都是16位。
如果 乘数是8位，结果为16位
如果 乘数是16位，结果为32位

乘数一个如果为16位就在AX中。 如果是8位就默认在AL中
另一个乘数在另一个寄存器，或者内存单元中

结果： 如果结果为16位，就放在AX中，如果结果为32位，AX存低16位。DX存高16位

例如

byte 为1个字节，所以取ds:[0]中的1个字节（8位）为乘数，
另一个乘数则为 al
因为乘数为8位，所以结果存在AX = al *(ds:[0])

word 为1个字，所以取ds:[0]中的1个字（16位）为乘数，
另一个乘数则为 ax
因为乘数为32位，所以结果存在AX和DX中

10.10 参数和结果传递问题 & 10.11批量数据传递问题 &12寄存器冲突问题

  1. 一般来说 参数和结果保存在寄存器中。
  2. 如果大批量传递数据，可以在寄存器中存批量数据的首地址，利用cx或者计算长度等方式，拿到数据
  3. 如果子程序中也用到外面的用到的寄存器怎么办？ 例如子程序改变了dx寄存器，而外面dx寄存器保存了其他值。
    1. 一般在子程序中，先用栈来保存当前寄存器的值。
    2. 当程序结束后，在恢复之前寄存器的值。
    3. 这样就保证了，子程序和父程序寄存器冲突的问题了。

8.5 指令要处理的数据有多长

8086CPU的指令，可以处理两个尺寸的数据，分别为byte和word, 分别为8位和16位

  1. 通过寄存器名称来指名处理数据的尺寸
    例如：
    mov ax, 1 处理为16位， 为一个字
    mov al, 1 处理为8位， 为一个字节
    mov al, ds:[0] 一个字节

  2. 在没有寄存器名称的情况下，用操作符 X ptr指名内存单元的长度， X在汇编指令中可以为word活byte
    以下指令用word ptr指令访问内存单元是一个字:
    mov word ptr ds:[0],1
    inc word ptr [bx]
    以下指令用byte ptr指令访问内存单元是一个字节
    mov byte ptr ds:[0],1

  3. 其他情况
    有些指令默认访问的是字节还是字单元，比如 push[1000h] 就不用指明是字还是字节单元，因为push只能进行字操作

8.7 div指令

div是除法指令，使用div做除法的时候应该注意一下几个问题

  1. 除数： 有8位和16位两种
  2. 被除数：默认放在AX或 DX和AX 中，
    1. 如果除数为8位，则被除数为16位。默认在AX中
    2. 如果除数为16位，则被除数在DX和AX中， AX中放低16位 DX放高16位
  3. 结果：
    1. 如果除数为8位， 则AL为商 AH为余数
    2. 如果除数为16位，则AX为商 DX为余数

div 操作的值在哪里，被除数多少位，都掌握在除数手中

8.8 伪指令dd

db: 定义字节类型数据
dw: 定义字类型数据
dd: 定义两个字类型的数据(double word)

8.9 dup

dup是个操作符，在汇编语言中同db、dw、dd等一样，也是由编译器识别处理的符号，需要配合db、dw、dd使用，进行数据的重复

db 3 dup (0) 定义了是3个字节，他们都是0 相当于db 0, 0 ,0
db 3 dup (0,1,2) 定义了是9个字节，他们都是0 相当于db 0, 1 ,2, 0, 1 ,2, 0, 1 ,2

3.6 栈

栈是一种数据结构，分为push（入栈）和pop（出栈）操作，以先进后出的形式操作数据，如下图所示

3.7 CPU提供的栈机制

现金的CPU都有栈的设计，8086CPU也不例外，8086CPU提供相关的指令来以栈的形式访问内存空间，两个基本操作为push（入栈）、pop(出栈）比如：
push ax表示将寄存器ax的数据送入栈中。
pop ax表示从站定取出来数据送入ax中。
8086中出栈入栈都是以字为单位的。

CPU要知道当前指令的所在位置，就要记录栈底地址和偏移量，8086CPU有两个寄存器，分别为SS寄存器和SP寄存器。在任何时刻SS:SP指向栈顶元素。
push和pop执行在执行时，CPU从SS和SP中得到栈顶地址。

push ax的执行由以下两步完成。
1）SP=SP-2，SS:SP指向当前栈顶前面的单元，以当前栈顶前面的单元为新栈顶（为什么是-2因为CPU操作都是以字为单位）
2）将ax的内容送到SS:SP指令指向的内存单元处，此时SS:SP指向新的栈顶（参考3.1字的存储）

入栈时，栈顶从高地址向低地址方向增长

3.8 栈顶超界的问题

当我们进行pop和push的时候，会有一种临界值。
就是当pop空栈的时候，sp依然会+2， push满栈的时候，sp依然会-2。这样赋值会造成栈空间意外的内存被改动，这就是我们说的栈顶超界的问题。在8086CPU中需要自己控制。

3.10 栈段

我们将内存空间当栈使用，以栈的方式进行访问，这段空间就可以成为栈段
如果我们将10000-1FFFFF这段空间当做栈段，初始状态栈是空的，SS=1000，SP=?
分析：
栈最底部的字单元地址为1000:FFFE,所以当第一个元素pop的时候，SP= FFFE + 2 = 10000, SP为16位，所以SP=0;

在从另一个角度去操作。当栈快满的时候，SP也为0（1000:2时ps = ps - 2),这样，如果再次亚展，栈顶将环绕，覆盖原来的内容，所以一个栈的最大内容为64KB
